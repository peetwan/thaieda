### Defect P1 — Insight Overflow (CRITICAL)
- **Files**: `src/thaieda/insight/__init__.py` (generate_insights) + `src/thaieda/report/__init__.py`
- **Problem**: `generate_insights()` ไม่มี cap สร้าง insight จาก quality + anomaly + distribution + correlation ทุกอันโดยไม่จำกัด
- **Evidence**:
  - aps-failure: 679 insights (391 warning + 288 info) ← รับไม่ได้
  - bike-sharing: 56 insights (26 warning + 30 info)
  - superstore: 56 insights (37 warning + 19 info)
  - online-shoppers: 49 insights (20 warning + 29 info)
  - absenteeism: 39 insights (17 warning + 22 info)
- **Fix**: 
  1. เพิ่ม parameter `max_insights: int = 30` ใน `generate_insights()`
  2. หลังจาก sort ตาม severity → ตัดให้เหลือ max_insights โดยเก็บ critical/warning ก่อน info
  3. สำหรับ anomaly overflow (>50 anomalies) ให้แสดง top 20 พร้อม "แสดง 20 จาก 362 รายการ"
  4. ส่วน Executive Summary ต้องบอกว่ามี insights ทั้งหมดกี่ข้อ แต่แสดงเพียง top 30
- **Target**: aps-failure ≤ 30 insights, superstore ≤ 30, online-shoppers ≤ 30
- **Constraint**: 
  - ห้ามตัด critical insights เลย (แสดงทั้งหมด)
  - warning ตัดได้แต่ต้องเก็บ ≥ 50%
  - info ตัดได้เต็มที่
  - ห้ามเปลี่ยน InsightSummary dataclass (เพิ่ม field total_generated ได้เพื่อบอกว่าตัดไปกี่ข้อ)

### Defect P2 — HTML Bloat: Charts & Tables Cap (HIGH)
- **File**: `src/thaieda/report/__init__.py` + `src/thaieda/viz/__init__.py`
- **Problem**: 
  - bike-sharing: 3.7MB, 61 charts, 30 tables → โหลดช้ามาก
  - aps-failure: 2.3MB, 208 tables → browser hang
- **Evidence**:
  - bike-sharing 61 base64 charts → 3.7MB HTML
  - aps-failure 208 tables (171 column details + 362 anomaly rows) → 2.3MB
- **Fix**:
  1. เพิ่ม `_MAX_CHARTS_PER_REPORT = 40` ใน report module — ถ้า charts > 40 ให้แสดงเฉพาะ top 40 ที่สำคัญที่สุด (prioritize anomaly/quality/insight charts ก่อน distribution charts)
  2. เพิ่ม `_MAX_TABLE_ROWS = 50` — ตารางที่เกิน 50 แถว ให้แสดง top 50 พร้อม "แสดง 50 จาก N แถว" + download CSV link placeholder
  3. Column Details: ถ้า > 60 columns ให้แสดง summary table แทน individual cards
  4. Anomaly table: ถ้า > 50 anomalies ให้แสดง top 50 + collapsed section สำหรับที่เหลือ
- **Target**: HTML < 2MB สำหรับทุก dataset
- **Constraint**: ห้ามลบ section สำคัญ (Priority Actions, Overview, Key Insights) เหลือแค่ตัด items ภายใน section

### Defect P3 — Performance: Wide & Tall Data (HIGH)
- **Files**: `src/thaieda/insight_engine/__init__.py` + `src/thaieda/viz/__init__.py`
- **Problem**:
  - aps-failure 171 cols × 16K rows = 491 seconds (เกินไป)
  - online-retail 541K rows = 397 seconds (เกินไป)
- **Evidence**:
  - aps-failure: 490.88s, 16,000 rows, 171 cols
  - online-retail: 396.70s, 541,909 rows, 8 cols
- **Fix**:
  1. `insight_engine`: ถ้า cols > 100 ให้ sample 50% ของ measures/breakdowns แทนการลองทั้งหมด
  2. `viz`: ถ้า cols > 50 ให้ skip scatter_matrix (เพราะ O(n²) charts)
  3. `viz`: ถ้า cols > 30 ให้ skip correlation_heatmap (O(n²) memory)
  4. `_detect_strong_correlations`: ใช้ pandas `.corr()` matrix (vectorized) แทน pairwise loops
  5. `_detect_outlier_insights`: ใช้ z-score vectorized แทน row-by-row
- **Target**: aps-failure < 120s, online-retail < 120s
- **Constraint**: ห้ามเปลี่ยน API signature หรือ output format

### Defect Q1 — Clean: High-NA Column Handling (MEDIUM)
- **File**: `src/thaieda/clean/__init__.py`
- **Problem**: `handle_missing_values` ไม่ handle numeric column ที่มี > 40% missing แตกต่างจาก column ที่มี < 40%
- **Evidence**: beijing-pm25 มี pm2.5 column ที่มี NA ~40% แต่ถูก fill แทนที่จะ flag
- **Fix**:
  1. ใช้ `missing_ratio = series.isna().sum() / len(series)`
  2. ถ้า missing_ratio > 0.40 → สร้าง `CleaningResult` ที่บอกว่า "คอลัมน์นี้มี missing > 40% — แนะนำให้ drop หรือ impute ด้วย domain knowledge"
  3. ถ้า missing_ratio > 0.80 → flag เป็น "mostly_missing" และ skip fill
  4. Default strategy ("flag") สำหรับ high-NA ให้สร้าง flag column `_<col>_was_missing` แทน fill
- **Target**: beijing-pm25 pm2.5 → แสดง "mostly_missing" warning แทน fill
- **Constraint**: ห้าม breaking existing tests สำหรับ normal-NA (< 40%) columns

### Defect Q2 — Report: Insight Section Shows All Items (MEDIUM)
- **File**: `src/thaieda/report/_template.py`
- **Problem**: Template แสดง insight_items ทั้งหมด ไม่มี pagination/collapse
- **Evidence**: aps-failure Key Insights แสดง 391 warning + 288 info = 679 items ในหน้าเดียว
- **Fix**:
  1. แสดงเพียง top 20 insights ใน Key Insights section
  2. เพิ่ม collapsible `<details>` tag สำหรับ insights ที่เหลือ: "แสดงอีก N ข้อ..."
  3. แสดง count badge: "Warning: N | Info: M" เหมือนเดิม แต่ items ด้านล่างถูกตัด
- **Target**: Key Insights section แสดง ≤ 20 items + collapse สำหรับที่เหลือ
- **Constraint**: ห้ามเปลี่ยน context dict structure — เปลี่ยนแค่ template rendering
