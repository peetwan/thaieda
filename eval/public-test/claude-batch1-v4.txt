## BUG TO FIX: BD-1 — Regex backreference `\1` broken on pandas 3.x Arrow backend

pandas 3.0.3 uses `string[pyarrow]` as default string dtype. Arrow's regex engine does NOT support `\1` backreferences. This causes "Invalid regular expression: invalid escape sequence: \1" errors in cleaning functions.

### Files to fix:

1. **src/thaieda/clean/__init__.py line 355** — `fix_tone_mark_stacking`:
   Current: `vectorized=lambda s: s.str.replace(_TONE_STACK_RE.pattern, r"\1", regex=True)`
   Fix: Use callable instead of `\1` string:
   ```python
   vectorized=lambda s: s.str.replace(_TONE_STACK_RE.pattern, lambda m: m.group(1), regex=True)
   ```

2. **src/thaieda/clean/__init__.py line 331** — `_vec_fix_repeated`:
   Current: `fixed = work.str.replace(pattern, lambda m: m.group(1) * max_repeat, regex=True)`
   This already uses a callable — check if it works. If it fails too, cast to object dtype first.

3. **Check ALL other `.str.replace(pattern, r"\1", regex=True)` patterns** in the entire src/thaieda/ directory. Any that use `\1`, `\2`, `\g<1>` as replacement string need to be converted to callables.

4. **Also check**: The `s.astype(str)` call in `_apply_str_transform` (line 115) may produce Arrow string dtype. Consider casting to `object` dtype explicitly where regex is used: `s.astype(object)` instead of `s.astype(str)`.
