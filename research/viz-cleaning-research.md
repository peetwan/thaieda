### Ridgeline plots & violin plots — ทางเลือกแทน histogram สำหรับ distribution หลายกลุ่ม

Ridgeline plot (joy plot) แสดง density distribution ของหลายกลุ่มซ้อนกันในแนวนอน เห็นการเปลี่ยนแปลง shape ระหว่างกลุ่มได้ชัดกว่า histogram แบบ separate ส่วน violin plot รวม box plot + kernel density ไว้ในรูปเดียว เหมาะกับ distribution ที่ไม่ใช่ normal (bimodal, skewed) — ปัญหาที่พบบ่อยในข้อมูลไทย เช่น อายุ/รายได้/ความยาวข้อความ ThaiEDA ใช้ histogram + box plot อยู่ ต่อยอดให้ auto-select violin เมื่อตรวจพบ bimodal distribution จะเพิ่ม insight ได้ดี

### stylecloud — word cloud สวยขึ้น + ทางเลือกแทน wordcloud library

### ftfy — แก้ mojibake และ encoding mix-up อัตโนมัติ

### ydata-profiling vs Sweetviz — data quality profiling ที่ ThaiEDA ต้องเรียนรู้

ydata-profiling (เดิม pandas-profiling) สร้าง HTML report แบบ inline แสดง per-column stats, correlations, missing matrix, warnings พร้อม actionable flag (high cardinality, skewed, constant) Sweetviz เน้น dataset comparison + target association ด้วย visualization ที่สวยและ compact จุดที่ ThaiEDA ยังขาด: auto-flag severity (critical/warning/info), correlation heatmap ที่กรองเฉพาะ strong correlation, และ dataset diff view สำหรับเปรียบเทียบข้อมูลก่อน/หลัง cleaning — น่าเอามาเป็น UX reference สำหรับ report layout รุ่นถัดไป

### missingno — visualize รูปแบบ missing data ระดับ matrix และ nullity correlation

missingno ให้ 4 view หลัก: matrix plot (เห็นตำแหน่ง missing เป็นแถบขาว), bar chart (จำนวน non-null ต่อ column), heatmap (nullity correlation ระหว่าง column -1 ถึง 1) และ dendrogram (cluster column ที่ missing pattern เหมือนกัน) nullity correlation สำคัญเพราะบอกว่า missing เป็นแบบ MNAR/MAR/MCAR — เช่น ถ้า column A หายเมื่อ column B มีค่า แปลว่าไม่ใช่สุ่ม ThaiEDA ตอนนี้มี quality checks แต่ไม่ได้ visualize pattern ของ missing การเพิ่ม matrix plot + nullity heatmap จะทำให้ user เห็นปัญหาที่ตัวเลขอย่างเดียวมองข้าม และช่วยตัดสินใจว่าควร impute หรือ drop

Source: https://www.nb-data.com/p/how-to-visualize-missing-data-patterns

### WCAG color contrast + matplotlib dark_background — ทำให้ dark theme report ของ ThaiEDA ผ่าน accessibility

WebAIM Million 2024 พบว่า 83.6% ของเว็บยังพิ่ม contrast ไม่ผ่าน WCAG สำหรับ dark theme ต้องใช้ contrast ≥4.5:1 (normal text) หรือ ≥3:1 (large text/UI) matplotlib มี `plt.style.use('dark_background')` built-in ที่สลับ text/border เป็นขาว แต่ default palette ยังไม่รองรับ colorblind แนะให้ ThaiEDA ใช้ colorblind-safe palette (viridis/cividis) + ตรวจ contrast ของ insight text กับ dark background ใน HTML report เพื่อให้ user อ่านได้ทั้งตาปกติและ colorblind
